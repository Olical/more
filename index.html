more

A framework for LESS

Incorperating normalisation, a customisable grid system, vendor prefixes and other useful mixins such as #more .circle(10px).

You will probably learn more from checking out the source but this page should get you going. More is a framework written in LESS, an intermediate language for CSS, that can also be used with any normal CSS based project. This is because precompiled versions are provided in the repository. It allows you to forget about resets, normalisation, grid systems and vendor prefixes by handling it all for you.

Normalisation

The normalisation is implemented via an adapted normalize.css, a normalise.less if you will. It helps make all browser behave a bit better by reducing the quirks caused by some of their default styles.

I really recommend looking through the source to see the kind of things it fixes.

Mixins

The mixins are probably my most loved part of more. Some help you create unfamiliar shapes such as circles and other take out the need for remembering and writing vendor prefixes.

These are split into two categories, helpers and vendor prefixes. Helpers help with sometimes difficult tasks and the vendor prefixes prefix your styles with the correct vendor attributes such as webkit and moz. Heres a quick example of using the vendor prefix mixins.
